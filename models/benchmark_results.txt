PROMPT CACHING IMPACT:
  The system prompt (tactical doctrine, unit roster, examples) is ~600-1000 tokens.
  With --cache-reuse 256, llama.cpp caches the KV state for the shared prefix.
  First request: full prompt processing (~34s on SSE4)
  Subsequent requests: skip cached prefix → only process user message (~6-9s)
  Speedup: ~4-5× on repeated calls with same system prompt.
  On AVX2 Linux server: expect ~1-2s cached responses.
